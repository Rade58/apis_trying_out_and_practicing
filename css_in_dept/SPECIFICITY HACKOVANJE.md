ZA POCETAK MOGU POSMATRATI SLUCAJ, KADA SE ELEMENTIMA, DODAJE KLASA *.disabled* (PREDPOSTAVLJAM DA SE TOM KLASOM DOVODE NEKI STILOVI, KOJI BI DISABLE-OVALI NEKO DUGME, NA PRIMER DA ELEMENTU ZADAJU NEKU BLEDJU BOJU TEKSTA I POZADINE) (NARAVNO TO BI BIO SLUCAJ, KADA SE DUGME PRAVI OD NA PRIMER PARAGRAFA (STO JE KAKO KAZE ESTELLE, VEOMA IDIOTSKI))

MOZDA SE ELEMENT (NESTO STO PREDSTAVLJA, NEKO DUGME (KAO STO REKOH, MOZDA NAPRAVLJENO OD PARAGRAFA)) STVARNO DISABLE-UJE, TAKO STO SE ZADA **pointer-events: none;**

NAIME, DEVELOPERI ILI DIZAJNERI, NA PRIMER, U SLUCAJU KADA JE ELEMENT DISABLE-OVAN (KADA IMA KLASU .disabled), ZELE DA NA OSNOVU TOGA, PROMENE IZGLED KURSORA DA BI ON BIO DEFAULT OBLIKA I TADA OBICNO KORISTE *!important* KEYWORD

KORISTE GA DA NE BI MORALI VODITI RACUNA O SPECIFICITY-JU, I DA IM STILOVI, POMENUTE .disabled KLASE NE BI BILI OVERRIDEN, KAO POSLEDICA CASCADE-A

KAKO BI POKAZAO NA STA MISLIM, KREIRACU PRIMER, SA SLEDECA DVA HTML ELEMENTA

```HTML

<!-- OVO DUGME, PORED KLASE, KOJA MU DA JE POINTER OBLIK KURSORA (.btn), IMA I KLASU .deactivate KOJA BI TREBALA
DA DISABLE-UJE ILI DEAKTIVIRA TAJ ISTI STIL, KAKO BI KURSOR IMAO DEFAULT OBLIK-->
<!-- MEDJUTIM KADA POGLEDAM CSS, VIDECU DA TA KLASA ZA cursor: default NEMA !important KEYWORD -->
<p class="btn deactivated" style="border: darkmagenta solid 2px; display: inline-block">
    Pritisni
</p>

<!-- OVO DUGME, PORED KLASE, KOJA MU DA JE POINTER OBLIK KURSORA (.btn), IMA I KLASU .disabled KOJA BI TREBALA
DA DISABLE-UJE ILI DEAKTIVIRA TAJ ISTI STIL, KAKO BI KURSOR IMAO DEFAULT OBLIK-->
<!-- MEDJUTIM KADA POGLEDAM CSS, VIDECU DA TA KLASA ZA cursor: default ZAISTA IMA !important KEYWORD -->

```

```CSS

/*KLASE ZA DISABLE-OVANJE*/

.disabled {   /* SPECIFICITY 0-1-0 (ZA DONJI STIL NIJE BITNO ZBOG !important KEYWORDA, KOJE DAJE MAKSIMALNI SPECIFICITY KLASI U SLUCAJU KONKRETNOG STILA)*/
    cursor: default !important;     /* SPECIFICITY 1-0-0-1-0 */
}

.deactivated {              /*SPECIFICITY  0-1-0 */
    cursor: default;
}

/* PARAGRAF DUGME TREBA OBICNO DA IMA  */

p.btn {                 /* SPECIFICITY 0-1-1 */
    cursor: pointer;
}

```

KADA OSMOTRIM, OBA DUGMETA, ODNOSNO, AKO PREDJEM KURSOROM, PREKO JEDNOG, PA PREK ODRUGOG, SHVATICU DA JE SAMO DRUGO DUGME DISABLE-OVANO, ODNOSNO ONO DUGME, KOJE IMA KLASU **.disabled**

### KAKO HACK-OVATI SPECIFICITY **.deactivated {...}** SELEKTORA, DA BI DOBIO VECI SPECIFICITY, KOJI BI BIO VECI OD SPECIFICITY-JA **p.btn {...}** SELEKTORA, ALI S TIM STO PRI TOME, **NE SMEM** UPOTREBITI **!importan** KEYWORD

PA TO SAM MOGAO URADITI TAKO STO UMESTO DA IMAM SELEKTOR SASTAVLJEN OD JEDNOG .deactivated CLASS SELECTOR-A (OVAKO &nbsp;&nbsp;&nbsp;&nbsp; --> &nbsp;&nbsp;&nbsp;&nbsp; .deactivate), USTVARI IMAM KOMBINOVANI SELECTOR, KOJI IMA NEKOLIKO CLASS SELECTORA (OVAKO &nbsp;&nbsp;&nbsp;&nbsp; --> &nbsp;&nbsp;&nbsp;&nbsp; .deactivated.deactivated)


### HACKING SPECIFICITY WITH IDs

PRVO CU OBZNANITI, JEDAN REAL-LIFE EXAMPLE

```HTML

<!-- NAPRAVIO SAM KONTEJNER I NESTOVAO THIRD PARTY WIDGET U NJEGA -->
<div id="thirdPartyWidget" style="width: 480px; height: 248px; border: tomato solid 2px;">   <!--MOJ KONTEJNER-->

    <div id="TheirWidget" style="width: 100%; height: 100%"></div>  <!-- NEST-OVANI WIDGET -->

</div>

```

NAIME, POSTOJI MOGUCNOST, DA AUTORI NEKOG 3rd PARTY API-A (KOJI KORISTIM), DEFINISAJUCI SVOJ WIDGET, NJEMU JESU ZADALI ID, I DA SU DEFINISALI PLAVI BACKGROUND COLOR

A TEMA MOG SAJTA JESTE TAKVA, DA MENI NIKAKO NE ODGOVARA, POMENUTI BLUE, BACKGROUND COLOR

A ONI SU, TAKODJE STAVILI !important PORED, POMENUTOG STILA

OVKAO

```CSS

/* WIDGET-OV CSS FAJL: */

#TheirWidget {
    background-color: blue !important;
}


/* MOJ CSS FAJL: */
/* A OVO JE ID SELEKTOR, KOJI SAM JA ZADAO WIDGET-U, ODNOSNO, PRE NEGO STO SAM ODLUCIO DA UVEDEM
WIDGET JA SAM NJEGOVOM CONTAINERU ZADO ID */

#thirdPartyWidget {
    background-color: #BADA55;       /* JASNO JE DA AKO POSTOJI GORNJI SELECTOR, DA ZAISTA NE POSTOJI NISTA */
}                                   /* STO BIH MOGAO URADITI, KAKO BI POVECO SPECIFICITY OVOG SELECTORA */

```

POMENUTO (STAVLJANJE !important, ALI I ZADAVANJE id-JA ELEMENTU) SE NIKAD NE TREBA RADITI, PRILIKOM KREIRANJA THIRD PARTY WIDGET-A (KAKO ESTELLE KAZE U SALI, "DON'T BUY THIR STOCK OPTIONS")

STA NE SMEM U RADITI, A STA BI TREBALO BITI OCITO

> PA NE SMEM UKLONITI CELI NJIHOV SELEKTOR, JER NA OSNOVU TOG ID-JA SU VEROVATN OSELEKTOVANI I DRUGI FEATURE-I, KOJI SU DOSL IS NJIHOVIM WIDGET-OM

STA RADITI U OVAKVOJ SITUACIJI

**MOGU I JA KORISTITI *!important* KAKO BI OVERRIDE-OVAO, AUTOROV STIL** ALI KAO STO SAM REKAO, NE ZELIM DA KORISTIM *!important*

**OVO JE NAJBOLJE RESENJE:**

PA MOGU UKLONITI, NJEGOVO *!important*, I UMESTO TOGA FORMIRATI KOMBINOVANI SELECTOR, SASTAVLJEN OD DVA ISTA ID SELECTOR-A, KOJI JE, KAO STO SAM REKAO, AUTOR ZADAO SVOM WIDGET-U

OVAKO

```CSS
/* WIDGET-OV CSS FAJL: */
/* UKLANJAM !important I MODIFIKUJEM SELECTOR, KAKO BI SE SASTOJAO OD DVA ISTA ID SELECTOR-A */

#TheirWidget#ThirWidget {           /* SPECIFICITY 2-0-0 */
    background-color: blue;
}

/* MOJ CSS FAJL: */
/* ZA SADA NEKA OSTANE KAO STO JE I BILO, U POGLEDU MOG CSS FAJLA */

#thirdPartyWidget {                 /* SPECIFICITY 1-0-0 */
    background-color: #BADA55;
}

```

SADA DA KAZEM, IMAM DOBRU *POLAZNU TACKU*

ODNOSNO HOCU DA KAZEM, AKO BUDEM ZELEO DA PROMENIM BACKGROUND COLOR WIDGET-A, TO CU RADITI UZ POMOC MOG ID-JA

A NECU NARUSITI NISTA U POGLEDU AUTOROVIH STILOVA, KOJI ZAVISE, ODNOSNO CIJA DEFINICIJA ZAVISE OD ThirWidget id-JA (ODNOSNO CIJA CSS SELEKCIJA ZAVISI OD TOG id-JA)

JOS JEDNA BITNA STVAR:

**KADA KORISTIM, PRIKAZANE VISETRUKE ID ILI CLASS SELEKTORE ( *#id#id {...}* ILI *.class.class {...}* ), POTREBNO JE DA TO SVE BUDE ISPRACENO KOMENTARIMA U CODE-U, JER BILO KO DRUGI BI POMISLIO DA JE, VELIKA GRESKA, ONO STO SAM NAPISAO**

KOMENTAR BI MOZDA TREBAO DA IZGLEDA OVAKO:

```CSS

#thirdPartyWidget#thirdPartyWidget {    /* ADDING SPECIFICITY */
    background-color: #BADA55;
}

```

AA STO SE TICE OVOG MOG PRIMERA, OVIM GORE SAM POSTIGAO DA DOBIJEM, MOZ ZELJENI BACKGROUND COLOR, AUTOROVOG WIDGET-A (ALI KONACNU 'ODLUKU' JE U OVOM SLUCAJU NA KRAJU IMAO CASCADE)

ZASTO?

PA POMENUTA DVA SELECTORA IAMJU ISTI SPECIFICITI, ALI MOJ JE DEFINISAN 'NA KRAJU VODOPADA'

```CSS

#TheirWidget#ThirWidget {           /* SPECIFICITY 2-0-0 */
    background-color: blue;
}


#thirdPartyWidget {                 /* SPECIFICITY 2-0-0 */ /* ALI CASCADE OMOGUCAVA OVOM SLECTORU, DA SE NJEGOVI STILOVI, NA KRAJU APLICIRAJU*/
    background-color: #BADA55;
}

```

### HACK IN CASE OF EMERGENCY

PRVO CU DEFINISATI, JEDAN SELECTOR


*OVO STO SAM DEFINISAO, NIKADA NE TREBA RADITI, SEM AKO POSTOJI NESTO STO MORAM OVERIDE-OVATI (SPOLJNI MARKUP, ODNOSNO SPOLJNI CSS, KOJI DOLAZI SA NEKOG DRUGOG SERVERA)
