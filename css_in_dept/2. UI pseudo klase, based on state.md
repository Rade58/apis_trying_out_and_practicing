# USER INTERFACE pseudo klase

## SELEKTOVANJE ELEMENAT U ODNOSU NA TRENUTNO STANJE UI-A

 &nbsp; &nbsp; **:enabled** &nbsp; &nbsp; SELEKTUJE BILO KOJI ENABLED input (KADA INPUT **NEMA** *disabled* ATRIBUT)

 &nbsp; &nbsp; **:disabled** &nbsp; &nbsp; SELEKTUJE BILO KOJI DISABLED input (KADA input **IMA** *disabled* ATRIBUT)

 &nbsp; &nbsp; **:checked** &nbsp; &nbsp; SELEKTUJE CHECKED INPUT (CHECKED CHECKBOX)

********************************
KAKO BI OVO POKAZAO NAJBOLJE JE DA SELEKTUJEM *label* ELEMENT, KOJI SE NALAZI POSLE *input*-A (NJEGOV JE SUSED), ALI TAJ *input* JESTE TIPA *checkbox* I TREBA DA BUDE CHECKIRAN DA BI SELEKTOVAO POMENUTI LABLE

```CSS
    input[type=checkbox]:checked + label {
        color: tomato;
    }
```

SADA KADA KORISNIK BUDE CHECKIRAO CHECKBOX, NA *label* KOJI SE NALAZI NAKON CHECKBOX-A, BICE PRIMENJENA BOJA, A KADA UNCHECKIRAM, BOJA VISE NECE BITI PRIMENJENA
********************************

### PSEUDO KLASE BITNE ZA **```<input type='number'>```**

 &nbsp; &nbsp; **:in-range** &nbsp; &nbsp; SELEKTUJE BILO KOJI input (type=number) AKO UNOS **JESTE** U OPSEGU

 &nbsp; &nbsp; **:out-of-range** &nbsp; &nbsp; SELEKTUJE BILO KOJI input (type=number) AKO UNOS **NIJE** U OPSEGU

 &nbsp; &nbsp; **:valid** &nbsp; &nbsp; SELEKTUJE BILO KOJI input AKO UNOS **JESTE**  VALIDAN

 &nbsp; &nbsp; **:invalid** &nbsp; &nbsp; SELEKTUJE BILO KOJI input AKO UNOS **NIJE** VALIDAN

NAJVAZNIJE ZA PRVE DVE PSEUDO KLASE, JESU SLEDECI ATRIBUTI input-A (type=number) :

> min    &nbsp;&nbsp;   max

AKO U INPUT, KOJI REPREZENTUJE SLEDECI input TAG

```HTML
<input type="number" max="18" min="6" step="2">

<style>

    input[type=number]:in-range {
        background-color: green;
    }

    input[type=number]:out-of-range {
        background-color: red;
    }

</style>
```

UNESEM BROJ KOJI JE MANJI OD 6 ILI VECI OD 18:

UNOS NECE BITI U RANGE-U I BICE PRIMENJEN ODGOVARAJUCI STIL

AKO UNESEN BROJ 10 (NA PRIMER) (ILI 18 ILI 6) UNOS CE BITI U RANGE-U I BICE PRIMENJEN ODGOVARAJUCI STIL

********************************

JOS NESTO STO JE VAZNO ZA **input** KOJI JE `type="number" JESTE NJEGOV ATRIBUT **step**

NAIME OVAJ ATRIBUT DEFINISE, ZA KOLIKO BROJEVA CE SE DODATI ILI ODUZETI OD TRENUTNE VREDNOSTI, KADA SE KLIKNE NA STRELICU TOG INPUT ELEMENAT (KARAKTERISTICNE STRELICE(IMA IH DVE) NA INPUT ELEMENTU, KOJI JE NUMBER TIPA)

TA VREDNOST, AKO SE IZOSTAVI; **step** ATRIBUT JE, PO DEFAULT-U **1**, A ONO STO JE TAKODJE MOGUCE, JESTE DEFINISATI BILO KOJU DECIMALNU VREDNOST ZA **step** ATRIBUT

ALI AKO STAVIM, NEKI DRUGI BROJ, NA PRIMER DECIMALNI, TO OSTAVLJA MNOGO PROSTORA ZA 'GRESKU', PRILIKOM UNOSA BROJA PUTEM TASTATURE (U BROWSERIMA, GDE JE TO DOZVOLJENO)

ZATO CU JA, SLEDECI INPUT ELEMENT, KOJEM SAM, KAO STO SE VIDI, DODAO **step** ATRIBUT SA VREDNOSCU **'0.41'**

```HTML

<input class="inpEl" type="number" min="4" max="8" step="0.41">

```

SELEKTOVATI PSEUDO KLASAMA: &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp;&nbsp; &nbsp; ***:in-range*** &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp;&nbsp; &nbsp; ***:out-of-range*** &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp;&nbsp; &nbsp; ***:valid*** &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp;&nbsp; &nbsp; ***:invalid*** &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp;&nbsp; &nbsp;

```CSS

    /* KADA JE UNOS U RANGE-U ZELIM DA TEKST (BROJ) BUDE OLIVE BOJE */

    /* KADA UNOS NIJE U RANGE-U, ZELIM DA TEKST BUDE TOMATO BOJE */

    /* KADA JE UNOS VALIDAN ZELIM DA BORDER BUDE OLIVE BOJE */

    /* KADA JE UNOS NEVALIDAN ZELIM DA BORDER BUDE TOMATO BOJE */

    input.inpEl:in-range {
        color: olive;
    }

    input.inpEl:out-of-range {
        color: tomato;
    }

    input.inpEl:valid {
        border: olive solid 2px;
    }

    input.inpEl:invalid {
        border: tomato solid 2px;
    }

```

NAJBOLJE BI BILO OBJASNITI OVAJ PRIMER, ODNOSNO INTERPRETIRATI, STA CE SE DOGODITI, KOD UNOSA RAZLICITIH BROJEVA

ONO STA JE SASVIM JASNO JESTE DA BILO KOJA CIFRA UNETA, A KOJA JE MANJA OD 4, A VECA OD 8, PROUZROKOVACE DA UNOS BUDE INVALIDAN, I DA BUDE OUT OF RANGE, A TO ZNACI TOMATO BORDER I TOMATO TEKST

ALI U JEDNOM DRUGOM VIDU, **step** JE 'POREMETIO' ODREDJENU KARAKTERISTIKU, KOJA BI NORMALNO BILA VALIDNA, DA JE **step** BIO **1**

NAIME, NAJBOLJE JE POSMATRATI MOGUCE, VALIDNE VREDNOSTI KOJE SE JAVLJAJU, PRI PRITISKANJU STRELICA

AKO PRITISNEM NA GORNJU STRELICU INPUT ELEMENTA (TIPA number) POJAVICE SE BROJ 4 (I TO JE PRVI VALIDNI BROJ)

NAIME, OVAKO CE IZGLEDATI PROMENA CIFRI (KOJE SU VALIDNE) ELEMENTA, NAKON PRITISKANJA, POMENUTE STRELICE

**4** &nbsp; &nbsp; &nbsp; &nbsp; **4.41** &nbsp; &nbsp; &nbsp; &nbsp;  **4.82** &nbsp; &nbsp; &nbsp; &nbsp; **5.23**  &nbsp; &nbsp; &nbsp; &nbsp;  **5.64**   &nbsp; &nbsp; &nbsp; &nbsp;  **6.05**  &nbsp; &nbsp; &nbsp; &nbsp;  **6.46**   &nbsp; &nbsp; &nbsp; &nbsp; **6.87**  &nbsp; &nbsp; &nbsp; &nbsp; **7.28** &nbsp; &nbsp; &nbsp; &nbsp; **7.69**

DAKLE, PROMENA JE POCELA OD 4, KOJEM JE DODATA VREDNOST step-A, PA ZATIM JE TOJ NOVOJ VREDNOSTI DODATA VREDNOST step-A ITD.

KAO STO VIDIM, MEDJU OVIM VALIDNIM CIFRAMA, NEMA BROJA 8, JER JE ON NEVALIDAN, IAKO JE ON MAKSIMALNA VREDNOST RANGE-A

BROJ 8 JE INVALIDAN ZBOG step-A, JER VREDNOST step-A NE DOZVOLJAVA DA SE 'DOSEGNE' BROJ 8

POKUSACU DA TASTATUROM UNESEM BROJ 8

BORDER JE POSTAO TOMATO (NEVALIDAN UNOS), ALI JE FONT OLIVE (UNOS JE U RANGE-U)

POKUSACU DA UNESEM SADA BILO KOJI BROJ, KOJI JE U OPSEGU IZMEDJU 4 I 8, ALI NIJE NI JEDAN OD PONUDJENIH (NA PRIMER, NEKA TO BUDE BROJ 6)

I OPET JE BORDER DOBIO TOMATO BOJU, STO ZNACI DA UNOS I U OVOM SLUCAJU JESTE NEVALIDAN, ALI TEKST JE OLIVE BOJE (STO ZNACI DA JE UNOS IN RANGE (U OPSEGU))

ZAPAZANJE:

> UNOS AKO JE OUT OF RANGE, ON JE I INVALIDAN

> TO NE MORA ZNACITI DA UNOS, KOJI JE IN RANGE, MORA UVEK BITI VALIDA

> GORNJI PRIMER, UPRAVO POKAZUJE, UNOS KOJI MOZE BITI IN RENGE, ALI JE NEVALIDAN, A STO JE IZAZVANO VREDNOSCU step ATRIBUTA (AKO TU VREDNOST, KORISNIKOV UNOS, NE ISPOSTUJE)

AKO MOJ UNOS, NIJE U email FORMATU, UNOS NECE BITI VALIDAN, I BICE PRIMENJEN ODGOVARAJUCI STIL

AKO MOJ UNOS JESTE U email FORMATU, UNOS CE BITI VALIDAN, I BICE PRIMENJEN ODGOVARAJUCI STIL ZA TO

> MOGUC JE I NEVALIDAN UNOS ZA input (type=number) ALI NEKI BROWSERI, KAO STO JE OPERA, CAK I NE OMOGUCUJU DA SE UNESE NA PRIMER LETTER U input (type=number)
>
> DRUGI BROWSERI TO DOZVOLJAVAJU ALI JE UNOS NEVALIDAN (I MOZE SE APLICIRATI STIL ZA NEVALIDAN UNOS, KORISCENJEM POMENUTE PSEUDO KLASE)

 &nbsp; &nbsp; **:required** &nbsp; &nbsp; SELEKTUJE BILO KOJI input AKO POMENUTI IMA *required* ATRIBUT (KOJI UTICE NA VALIDNOST (ODNOSNO, AKO OSTAVIM ELEMENT SA PRAZNIM UNOSOM, TAKAV UNOS JE NEVALIDAN, AKO input IMA required ATRIBUT, A AKO JE IZOSTAVLJEN required ATRIBUT, POMENUTI UNOS JE VALIDAN))

 ```CSS

    input:required {
        border-width: 5px;
    }

    /* SLEDECI CODE NEMA VEZE SA required ATRIBUTOM, ALI IPAK CU GA OSTAVITI OVDE */
    /* OVO MI JE KAO PODSETNIKA DA SE TREBAM POZABAVITI aria-OM */
    /* DAKLE, AKO ELEMENT IMA aria-required ATRIBUT SA VREDNOSCU true, TREBALO BI DA GA SEKTUJEM PUTEM
    ATTRIBUTE SELECTOR-A */
    input[aria-requred='true'] {
        border-width: 5px;
    }

    /* ONO STO SAM MOGAO, VEROVATNO JE I DOVRA PRAKSA, DEFINISATI ODJEDNOM STILOVE ZA OVA DVA SELECTORA
    TAKO STO BIH NAPISAO NA SLEDECI NACIN */

    input[aria-required="true"], input:required {
        border-width: 5px;
    }

 ```

 &nbsp; &nbsp; **:optional** &nbsp; &nbsp; SELEKTUJE BILO KOJI input, KOJI **NEMA** *required* ATRIBUT (AKO NESTO NIJE ZAHTEVANO (:required), ONO JE OPCIONO (:optional))

 > VAZNA STVAR: UPRAVO PRE INTERAKCIJE SA INPUT ELEMENTOM (TIPA number), KADA NISTA NIJE UNESENO, ODNOSNO NIJE PIPNUTA NI JEDNA STRELICA, UNOS CE BITI VALIDAN (:valid) (BICE APLICIRAN ONAJ STIL :valid PSEUDO KLASE, AKO POSTOJI), JER NISAM 'TRAZIO' DA UNOS BUDE REQUIRED
 > OVO UPRAVO ZNACI DA JE UNOS VALIDAN AKO JE PRAZAN A NIJE REQUIRED-OVAN
 > I ZNACI DA JE NEVALIDAN AKO JE PRAZAN I JESTE REQUIRED-OVAN

&nbsp; &nbsp; **:default** &nbsp; &nbsp; SELEKTUJE PRVENSTVENO input (type=checkbox|radio) AKO POMENUTI IMA *checked* ATRIBUT (TO MOZE DA BUDE I option KOJI IMA *selected* ATRIBUT)

&nbsp; &nbsp; **:read-only** &nbsp; &nbsp; SELEKTUJE input KOJI IMA *readonly* ATRIBUT (U TSKSV INPUT NIJE MOGUCE UNOSITI, ALI JE MOGUCE KOPIRATI TEKST IZ NJEGA); MEDJUTIM OVA PSEUDO KLASA NE BIRA SAMO input, VEC BILO KOJI ELEMENT KOJI NIJE MOGUCE EDIT-OVATI; AKO NA PRIMER POSTOJI **PARAGRAF KOJI IMA ATRIBUT *contenteditable="true"***, ON NECE BITI SELEKTOVAN, POMENUTOM PSEUDO KLASOM

&nbsp; &nbsp; **:read-write** &nbsp; &nbsp; SELEKTUJE EDITABLE ELEMENT (SVE VAZI SUPROTNO OD GORNJE PSEUDO KLASE) (DAKLE, AKO ELEMENT **NEMA** ATRIBUT *read-only* ONDA JE POTREBNO DA )

### PSEUDO KLASE, BAZIRANE NA STATE-U, KOJE SE KORISTE SA FORM ELEMENTIMA, ELEMENTA, A KOJE SU, JOS UVEK EKSPERIMENTALNE, ILI SU PROPOSED (PREDLOZENE PA SE CEKA NA NJIHOVO USVAJANJE)

&nbsp; &nbsp; **:placeholder-shown** &nbsp; &nbsp; SELEKTUJE PRVENSTVENO **input** ILI **textarea** AKO, NEKI OD POMENUTIH ELEMENATA IMA *placeholder* ATRIBUT (SA ODREDJENOM VREDNOSCU, KOJA NIJE PRAZAN STRING)

SLEDECA PSEUDO KLASE JE PREDLOZENA, I ONA JE UNDER CONSIDERATION

&nbsp; &nbsp; **:user-invalid** &nbsp; &nbsp;

NAJLAKSE JE MOGU OBJASNITI PUTEM PRIMERA

OPET JE REC O input ELEMENTU, KOJI JE TIPA 'number'

```HTML

<input class="numKlasa" type="number" min="4" max="8" value="10">

<!-- AKO DOBRO POGLEDAM OVAJ ELEMENT, MOGU SHVATITI DA JE ON ODMAH NEVALIDAN -->
<!-- A NEVALIDAN JE IZ RAZLOGA STO JE OUT OF RANGE -->
<!-- JER JE VREDNOST UNOSA DESET, ODNOSNO VREDNOST NIJE CEO BROJ IZMEDJU CETVORKE I OSMICE -->

<!-- TAKAV ELEMENT SE MOZE SELEKTOVATI PSEUDO KLASOM :invalid -->

<!-- ALI OVO JE 'NEVALIDNOST', U STARTU NAPRAVLJENA OD STRANE DEVELOPERA, A NE KORISNIKA -->

<!-- AKO ZELIM DA DEFINISEM STIL, KAKAV BI SE PRIMENIO, SAMO ONDA, KADA KORISNIK NAPRAVI NEVALIDAN UNOS -->
<!-- KORISTIO BI PSEUDO KLASU :user-invalid -->

```

```CSS

input.numKlasa:user-invalid {
    border: tomato solid 4px;
    color: tomato;
}

/* ONO STO OVDE NECU URADITI, JESTE DEFINISANJE STILA ZA NEVALIDNOST, KOJU PODRAZUMEVA :invalid PSEUDO KLASA
JER BI NA NEVALIDNOST, TADA UTICALA VREDNOST 10, KOJU JE DODAO DEVELOPER, A JA STILOVE ZA TAJ SLUCAJ NECU DA
DEFINISEM */

/* ALI KAO STO SAM REKAO, OVO JOS NE FUNKCIONISE, JER JE SAMO PROPOSAL */

```
